STAR WARS

An API Project Story

This is my first project that I am doing using JavaScript. By extension this is also the first project in my software engineering career. The project helped me to understand API's and helped me get more comfortable with Git.

The criteria for this project was to create a front-end wesbsite using an external API and to use a data base. At this stage I ended up mocking a server using JSON. The JSON server was used to access images as part of the proejct.

The API that was used was the Star Wars API or SWAPI for short. Obviously a huge Star Wars fan, and the best in the series being A New Hope and Empire Strikes Back. Doing a project based on Star Wars got me excited and gave me the motivation I needed.

The wesbite itseld is split into 5-tabs: People, Planets, Species, Vehicles, and Starships. In a nutshell when the button is pressed in any of the tabs, it will give information about that praticular person or planet. Each subsequent click will output a random character or planet or species.

A shortcoming of the API is that there are no pictures, and this is where the JSON server came in. By mokcing a JSON server POST requests was made to the server where it would store links to images. Postman was used to make the POST requests, and only a single JSON server was necessary for it to work.
